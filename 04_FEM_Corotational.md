## 为什么需要这一页

在最小 FEM 教学脚本里，我们先用了一个简单能量：

```math
\psi(F) = \frac12 \|F - I\|_F^2
```

它很适合入门，但有一个明显问题：

**纯旋转也会被它错误地当成变形。**

## `rotate` 案例说明了什么

在 `fem_triangle_demo.py` 的 `rotate` 案例里：

- 当前三角形只是整体旋转了 `90°`
- 没有拉伸
- 没有压缩
- 没有剪切

所以物理上应该认为：

- 没有真正弹性变形
- 不应该产生弹性能

但简单模型会给出很大的能量，因为它只看 `F` 和 `I` 的差别。

## co-rotational 的核心想法

一句话：

**先把旋转拿掉，再衡量真正的变形。**

也就是说：

- `F` 里面混着旋转和形变
- 我们不应该惩罚“整体旋转”
- 我们只应该惩罚“真正拉伸、压缩、剪切”的部分

## 教学版理解

可以把 co-rotational 视角理解成：

```math
\psi(F) \propto \|F - R\|^2
```

其中：

- `R` 是最接近 `F` 的旋转矩阵

于是：

- 如果 `F` 本身就是纯旋转
- 那么 `F = R`
- 所以能量为 `0`

这就比简单的 `\|F-I\|^2` 更合理。

## 在当前脚本里你能看到什么

运行：

```powershell
python D:\学习\编程学习\具身智能\具身智能\rope_demos\fem_triangle_demo.py --case rotate
```

你会看到：

- `simple psi(F)` 很大
- `co-rotational psi(F)` 为 `0`

这说明：

- 简单模型把旋转误判成了变形
- co-rotational 视角把纯旋转正确剥离掉了

## 这一页真正要记住的 4 句话

1. 纯旋转不是弹性变形。
2. 所以纯旋转不应该产生弹性能。
3. `\|F-I\|^2` 只是教学模型，不够物理正确。
4. co-rotational 的直觉是“去掉旋转，再衡量形变”。
